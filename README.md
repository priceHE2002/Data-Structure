# Data-Structure

本项目是基于 C++ 实现的数据结构与算法学习仓库，涵盖了线性表、栈与队列、字符串、树与二叉树、图、查找算法、排序算法等核心内容。适合数据结构与算法课程学习、考研复习以及面试准备。

每个 `.cpp` 文件都是**独立可编译运行**的完整程序，包含数据结构定义、基本操作实现和测试用的 `main` 函数。

## 📖 各模块详解

### 1. 线性表 (0-LinarList)

#### 1.1 顺序表 (SeqList.cpp)

**存储结构**：使用动态数组实现，包含 `data` 指针（指向数组首地址）、`length`（当前长度）、`MaxSize`（最大容量）。

| 函数 | 功能说明 |
|------|---------|
| `InitSeqList(List)` | 初始化顺序表，分配初始容量为 10 的数组空间 |
| `IncreaseSeqList(List, num)` | 动态扩容，将容量增加 `num` 个单元 |
| `SeqlistInsert(List, i, e)` | 在位序 `i`（从 1 开始）处插入元素 `e`，需将 `i` 之后的元素后移 |
| `SeqlistDelete(List, i)` | 删除位序 `i` 处的元素，需将 `i` 之后的元素前移 |
| `LocateElem(List, e)` | 按值查找，输出所有值为 `e` 的元素的位序 |
| `PrintSeqList(List)` | 打印顺序表中的所有元素 |

**核心特点**：
- 支持随机访问，时间复杂度 O(1)
- 插入/删除需要移动元素，平均时间复杂度 O(n)
- 采用动态扩容策略，避免静态数组容量不足

---

#### 1.2 带头结点的单链表 (LinkListWithHead.cpp)

**存储结构**：每个结点包含 `data`（数据域）和 `next`（指针域），头结点不存储实际数据，仅作为链表入口。

| 函数 | 功能说明 |
|------|---------|
| `InitLinkListWithHead(L)` | 初始化带头结点的单链表，创建头结点 |
| `GetElem(L, i)` | 按位查找，返回第 `i` 个结点的指针（头结点为第 0 个） |
| `LocateElem(L, e)` | 按值查找，返回第一个数据域为 `e` 的结点指针 |
| `length(L)` | 求链表长度（不含头结点） |
| `InsertNextNode(p, e)` | 在结点 `p` 之后插入新结点，数据为 `e` |
| `ListInsert(L, i, e)` | 在位序 `i` 处插入元素 `e`，调用 `GetElem` 找到前驱结点 |
| `InsertPriorNode(L, p, q)` | 在结点 `p` 之前插入结点 `q`，采用"后插+交换数据"技巧 |
| `ListDelete(L, i, e)` | 删除位序 `i` 的结点，并返回其数据值 |
| `DeleteNode(p)` | 删除结点 `p`，采用"偷梁换柱法"（复制后继数据后删除后继） |
| `DestroyList(L)` | 销毁整个链表，释放所有结点内存 |
| `PrintLinkList(L)` | 打印链表中所有元素 |
| `List_TailInsert_WithFunc(L)` | 尾插法建立链表（调用封装函数） |
| `List_TailInsert_WithOutFunc(L)` | 尾插法建立链表（不调用封装函数，手动处理指针） |
| `List_HeadInsert_WithFunc(L)` | 头插法建立链表（调用封装函数） |
| `List_HeadInsert_WithOutFunc(L)` | 头插法建立链表（不调用封装函数） |
| `ReverseLink(L)` | 逆置单链表，采用头插法思想逐个摘下结点重新插入 |

**核心特点**：
- 头结点简化了对第一个位置的操作，统一了空表和非空表的处理逻辑
- 头插法建立的链表与输入顺序相反，尾插法与输入顺序一致
- 插入/删除不需要移动元素，只需修改指针，时间复杂度 O(1)（已知前驱时）

---

#### 1.3 不带头结点的单链表 (LinkListWithoutHead.cpp)

**存储结构**：与带头结点的单链表相同，但第一个结点直接存储数据，没有额外的头结点。

| 函数 | 功能说明 |
|------|---------|
| `InitLinkListWithoutHead(L)` | 初始化不带头结点的单链表，`L` 初始为 `nullptr` |
| `ListInsert(L, i, e)` | 在位序 `i` 处插入元素 `e`。当 `i == 1` 时需特殊处理（修改头指针） |
| `List_HeadInsert_WithOutFunc(L)` | 头插法建立链表，每次新结点都插入到表头 |

**核心特点**：
- 第一个结点的操作需要特殊处理（修改头指针 `L`）
- 空表判断：`L == nullptr`
- 代码相对繁琐，但节省了一个头结点的空间

---

#### 1.4 双链表 (DoubleLinkList.cpp)

**存储结构**：每个结点包含 `data`（数据域）、`prior`（前驱指针）、`next`（后继指针），可双向遍历。

| 函数 | 功能说明 |
|------|---------|
| `InitDLinkList(L)` | 初始化双链表，创建头结点，`prior` 和 `next` 均为 `nullptr` |
| `InsertNextDNode(p, s)` | 在结点 `p` 之后插入结点 `s`，需修改 4 个指针 |
| `DeleteNextDNode(p)` | 删除结点 `p` 的后继结点，需处理前驱指针的更新 |
| `DestroyDLinkList(L)` | 销毁双链表，从前往后依次删除结点后释放头结点 |

**核心特点**：
- 支持双向遍历，查找前驱时间复杂度 O(1)
- 插入/删除需要修改更多指针（4 个），但逻辑清晰
- 适用于需要频繁前后移动的场景

---

#### 1.5 循环链表 (CLinkList.cpp)

**存储结构**：与单链表相同，但最后一个结点的 `next` 指向头结点，形成环。

| 函数 | 功能说明 |
|------|---------|
| `InitCLinkList(L)` | 初始化循环链表，头结点的 `next` 指向自身 |
| `isEmpty(L)` | 判断循环链表是否为空（头结点的 `next` 是否指向自身） |
| `isTail(L, p)` | 判断结点 `p` 是否为表尾结点（`p->next == L`） |

**核心特点**：
- 从任意结点出发都可以遍历整个链表
- 空表判断条件与单链表不同
- 适用于需要循环访问的场景（如约瑟夫环问题）

---

#### 1.6 静态链表 (StaticLinkList.cpp)

**存储结构**：使用数组模拟链表，数组元素包含 `data`（数据域）和 `next`（下一个元素的数组下标）。下标 `-1` 表示链表结束，`-2` 表示空闲结点。

| 函数 | 功能说明 |
|------|---------|
| `InitStaticLinkList(a)` | 初始化静态链表，所有结点的 `next` 设为 `-2`（空闲），头结点 `next` 为 `-1` |
| `length(a)` | 求静态链表长度 |
| `LocateElem(a, i, p)` | 找到位序为 `i` 的结点的数组下标 `p` |
| `InsertNode(a, i, e)` | 在位序 `i` 处插入元素 `e`。先找空闲位置，再修改指针完成插入 |

**核心特点**：
- 不需要指针，适合不支持指针的高级语言
- 需要手动管理空闲空间（查找 `next == -2` 的结点）
- 插入/删除只需修改数组下标，不需要移动元素

### 2. 栈 (1-Stack)

#### 2.1 顺序栈 (SqStack.cpp)

**存储结构**：使用数组实现，通过 `top` 指针（数组下标）指示栈顶位置。`top = -1` 表示栈空。

| 函数 | 功能说明 |
|------|---------|
| `InitSqStack(S)` | 初始化顺序栈，`top` 设为 `-1` |
| `isSqStackEmpty(S)` | 判断栈是否为空（`top == -1`） |
| `Push(S, e)` | 入栈，先判断栈满（`top == MaxSize - 1`），再将 `top` 加 1 后存入元素 |
| `Pop(S, e)` | 出栈，先判断栈空，再取出栈顶元素后将 `top` 减 1 |
| `GetTop(S, e)` | 读取栈顶元素，不出栈 |

**核心特点**：
- 实现简单，存取效率高
- 容量固定（可扩展为共享栈或链栈解决）
- 所有操作时间复杂度均为 O(1)

---

#### 2.2 链栈 (LinkStack.cpp)

**存储结构**：使用单链表实现，头结点为栈顶，所有操作在表头进行。

| 函数 | 功能说明 |
|------|---------|
| `InitLinkStack(S)` | 初始化链栈，创建头结点 |
| `Push(S, e)` | 入栈，采用头插法在栈顶插入新结点 |
| `Pop(S, e)` | 出栈，删除头结点后的第一个结点 |
| `GetTop(S, e)` | 读取栈顶元素（头结点后的第一个结点的数据） |

**核心特点**：
- 不存在栈满的情况（除非内存耗尽）
- 入栈/出栈相当于单链表的头插/头删，时间复杂度 O(1)
- 多个栈可以共享空间，提高效率

---

#### 2.3 括号匹配 (bracketCheck.cpp)

**算法思想**：使用栈检查表达式中的括号是否匹配。遇到左括号入栈，遇到右括号则与栈顶元素比较。

| 函数 | 功能说明 |
|------|---------|
| `InitLinkStack(S)` | 初始化链栈（辅助结构） |
| `Push(S, a)` | 左括号入栈 |
| `Pop(S, a)` | 右括号时出栈并比较 |
| `bracketCheck(str, length)` | 主算法：遍历字符串，检查括号匹配情况 |

**匹配规则**：
- `(` 与 `)` 匹配，`[` 与 `]` 匹配，`{` 与 `}` 匹配
- 栈空时遇到右括号 → 不匹配
- 栈顶括号与当前右括号类型不同 → 不匹配
- 遍历结束后栈非空 → 不匹配

---

#### 2.4 中缀转后缀表达式 (InfixToPostfix.cpp)

**算法思想**：使用栈将中缀表达式转换为后缀表达式（逆波兰式）。遇到操作数直接输出，遇到运算符根据优先级入栈或出栈。

| 函数 | 功能说明 |
|------|---------|
| `isOperator(c)` | 判断字符是否为运算符（`+`、`-`、`*`、`/`、`^`） |
| `getPriority(op)` | 获取运算符优先级：`+`/`-` 为 1，`*`/`/` 为 2，`^` 为 3 |
| `InfixToPostfix(infix, postfix)` | 主算法：扫描中缀表达式，输出后缀表达式 |

**转换规则**：
1. 操作数（数字/字母）直接加入后缀表达式
2. 左括号 `(` 直接入栈
3. 右括号 `)` 弹出栈内运算符直到遇到左括号
4. 运算符：弹出栈中优先级 **大于等于** 当前运算符的所有运算符，然后当前运算符入栈
5. 扫描结束后，将栈中剩余运算符依次弹出

**多位数处理**：连续的数字作为一个整体输出，数字之间用空格分隔。

### 3. 队列 (2-Queue)

#### 3.1 顺序队列 (SqQueue1.cpp)

**存储结构**：使用数组实现，通过 `front` 和 `rear` 指针分别指示队头和队尾。采用**牺牲一个存储单元**的策略区分队空和队满。

| 函数 | 功能说明 |
|------|---------|
| `InitSqQueue(Q)` | 初始化队列，`front = rear = 0` |
| `isQuereEmpty(Q)` | 判断队列是否为空（`front == rear`） |
| `enQueue(Q, e)` | 入队，先判断队满（`(rear + 1) % MaxSize == front`），再在 `rear` 位置存入元素，`rear` 循环后移 |
| `deQueue(Q, e)` | 出队，先判断队空，再取出 `front` 位置元素，`front` 循环后移 |
| `GetElem(Q, e)` | 读取队头元素，不出队 |
| `Length(Q)` | 求队列长度：`(rear + MaxSize - front) % MaxSize` |

**核心特点**：
- 牺牲一个单元区分队空和队满，避免使用计数器或标志位
- 采用循环队列解决"假溢出"问题
- 所有操作时间复杂度均为 O(1)

---

#### 3.2 循环队列（引入 size）(SqQueue2.cpp)

**存储结构**：与 SqQueue1 相同，但增加 `size` 变量记录当前队列长度，不使用牺牲单元策略。

| 函数 | 功能说明 |
|------|---------|
| `InitSqQueue(Q)` | 初始化队列，`front = rear = size = 0` |
| `isQuereEmpty(Q)` | 判断队列是否为空（`size == 0`） |
| `enQueue(Q, e)` | 入队，判断队满（`size == MaxSize`），存入元素后 `rear` 循环后移，`size++` |
| `deQueue(Q, e)` | 出队，判断队空，取出元素后 `front` 循环后移，`size--` |
| `GetElem(Q, e)` | 读取队头元素 |

**核心特点**：
- 使用 `size` 变量明确记录队列长度，逻辑更清晰
- 可以充分利用所有数组空间（不需要牺牲一个单元）
- 队满和队空判断更简单直接

---

#### 3.3 链队列 (LinkQueue.cpp)

**存储结构**：使用带头结点的单链表实现，`front` 指向头结点，`rear` 指向队尾结点。

| 函数 | 功能说明 |
|------|---------|
| `InitLinkQueue(Q)` | 初始化链队列，创建头结点，`front` 和 `rear` 均指向头结点 |
| `IsLinkQueueEmpty(Q)` | 判断队列是否为空（`front == rear`） |
| `enLinkQueue(Q, e)` | 入队，在队尾插入新结点，更新 `rear` 指针 |
| `deLinkQueue(Q, e)` | 出队，删除队头结点。若删除后队列为空，需将 `rear` 指向 `front` |
| `length(Q)` | 求队列长度，从队头遍历到队尾计数 |
| `DestroyLinkQueue(Q)` | 销毁链队列，从队头到队尾依次释放所有结点 |

**核心特点**：
- 不存在队满的情况（除非内存耗尽）
- 出队时需要特殊处理：当队列中只有一个元素时，删除后 `rear` 需要回退到 `front`
- 适合数据量不确定的场景

### 4. 字符串 (3-String)

#### 4.1 堆分配存储的串 (HString.cpp)

**存储结构**：使用动态分配的字符数组存储字符串，包含 `ch` 指针（指向字符数组）和 `length`（串长）。

| 函数 | 功能说明 |
|------|---------|
| `InitHString(S, maxLen)` | 初始化串，分配最大长度为 `MaxLen` 的字符数组空间 |
| `DestroyHString(S)` | 销毁串，释放字符数组内存 |
| `StrAssign(S, T, tLength)` | 赋值操作，将字符数组 `T` 的内容赋给串 `S` |
| `StrCopy(S)` | 复制操作，返回串 `S` 的副本 |
| `StrEmpty(S)` | 判空操作，判断串是否为空（`ch == nullptr`） |
| `StrCompare(S, T)` | 串的比较，先逐字符比较，若都相等则比较长度。`S > T` 返回 1，`S < T` 返回 -1，`S == T` 返回 0 |
| `SubString(Sub, S, pos, len)` | 求子串，返回串 `S` 从第 `pos` 个字符起长度为 `len` 的子串 |

**模式匹配算法**：

| 函数 | 算法说明 |
|------|---------|
| `Index_1(S, T)` | **朴素模式匹配（方案一）**：遍历主串 `S`，对每个位置检查是否匹配子串 `T`。时间复杂度 O((n-m+1)*m) |
| `Index_2(S, T)` | **朴素模式匹配（方案二）**：使用双指针 `i` 和 `j` 同时遍历主串和子串，不匹配时 `i` 回退到 `i-j+1`。与方案一等价 |
| `Index_3(S, T)` | **基于子串比较的模式匹配**：每次从主串中取长度为 `m` 的子串，与子串 `T` 比较。思路直观但效率较低 |
| `KMP_Index(S, T, next)` | **KMP 算法**：利用 `next` 数组避免不必要的回溯，时间复杂度 O(n+m)。本实现字符串下标从 0 开始 |

**KMP 算法特点**：
- `next` 数组记录了子串中每个位置的最长相等前后缀长度
- 当匹配失败时，根据 `next` 数组决定子串指针 `j` 的回退位置，主串指针 `i` 不回退
- 相比朴素算法，大大提高了匹配效率

### 5. 树与二叉树 (4-Tree)

#### 5.1 顺序存储二叉树 (SqBinaryTree.cpp)

**存储结构**：使用数组存储二叉树，对于下标为 `i` 的结点：左孩子下标为 `2*i+1`，右孩子下标为 `2*i+2`，双亲下标为 `(i-1)/2`。使用 `isEmpty` 标志位标记该位置是否有结点。

| 函数 | 功能说明 |
|------|---------|
| `InitSqBinaryTree(a)` | 初始化顺序二叉树，所有位置标记为空 |
| `InsertTreeNode(a, i, value)` | 在下标 `i` 处插入值为 `value` 的结点 |
| `HasLeftChild(a, i)` | 判断下标 `i` 的结点是否有左孩子 |
| `HasRightChild(a, i)` | 判断下标 `i` 的结点是否有右孩子 |
| `GetLeftChild(a, i, value)` | 获取下标 `i` 结点的左孩子的值 |
| `GetRightChild(a, i, value)` | 获取下标 `i` 结点的右孩子的值 |
| `GetParent(a, i, value)` | 获取下标 `i` 结点的双亲的值 |
| `IsLeaf(a, i)` | 判断下标 `i` 的结点是否为叶子结点 |
| `PrintSqBinaryTree(a)` | 打印顺序二叉树，显示每个结点及其孩子信息 |

**核心特点**：
- 适合存储完全二叉树，空间利用率高
- 非完全二叉树会浪费大量空间
- 父子结点关系通过下标计算，不需要指针

---

#### 5.2 链式存储二叉树 (LinkBiTree.cpp)

**存储结构**：每个结点包含 `value`（数据域）、`leftChild`（左孩子指针）、`rightChild`（右孩子指针）。

| 函数 | 功能说明 |
|------|---------|
| `InitBiTree(root, data)` | 初始化二叉树，创建根结点，数据为 `data` |
| `InsertNewNode(parent, data, child)` | 在 `parent` 结点后插入新结点。`child` 为 `'l'` 时插入左孩子，为 `'r'` 时插入右孩子 |
| `visit(p)` | 访问结点 `p`，打印其数据值 |
| `PreOrder(T)` | **先序遍历**：根 → 左子树 → 右子树 |
| `InOrder(T)` | **中序遍历**：左子树 → 根 → 右子树 |
| `PostOrder(T)` | **后序遍历**：左子树 → 右子树 → 根 |
| `treeDepth(T)` | 求树的深度，递归计算左右子树深度的最大值加 1 |
| `levelOrder(T)` | **层序遍历**：使用辅助队列，从上到下、从左到右逐层访问结点 |

**核心特点**：
- 适合存储非完全二叉树，空间利用率高
- 三种递归遍历算法是二叉树的基础，必须熟练掌握
- 层序遍历需要借助队列实现

---

#### 5.3 线索二叉树 (ThreadBiTree.cpp)

**存储结构**：在二叉链表的基础上增加 `ltag` 和 `rtag` 标志位。`tag = 0` 表示指针指向孩子，`tag = 1` 表示指针指向线索（前驱或后继）。

| 函数 | 功能说明 |
|------|---------|
| `InitThreadBiTree(root, data)` | 初始化线索二叉树，创建根结点，`ltag` 和 `rtag` 初始为 0 |
| `InsertNewNode(parent, data, child)` | 插入新结点，与链式二叉树相同 |
| `visit(q)` | 访问结点并建立线索：若左子树为空则建立前驱线索；若前驱的右子树为空则建立前驱的后继线索 |
| `CreateInThread(T)` | **中序线索化**：中序遍历二叉树，一边遍历一边线索化 |
| `CreatePreThread(T)` | **先序线索化**：先序遍历二叉树，一边遍历一边线索化。注意避免死循环（当 `ltag == 1` 时不递归左子树） |
| `CreatePostThread(T)` | **后序线索化**：后序遍历二叉树，一边遍历一边线索化 |

**中序线索二叉树遍历**：

| 函数 | 功能说明 |
|------|---------|
| `FirstNode(P)` | 找到以 `P` 为根的子树中，第一个被中序遍历的结点（最左下角） |
| `LastNode(P)` | 找到以 `P` 为根的子树中，最后一个被中序遍历的结点（最右下角） |
| `NextNode(P)` | 找到结点 `P` 的中序后继。若 `rtag == 1` 直接返回右线索；否则返回右子树的最左下角结点 |
| `PreNode(P)` | 找到结点 `P` 的中序前驱。若 `ltag == 1` 直接返回左线索；否则返回左子树的最右下角结点 |
| `InOrderThread(T)` | 对中序线索二叉树进行中序遍历（正序），不需要递归或栈 |
| `RevInOrderThread(T)` | 对中序线索二叉树进行逆序中序遍历 |

**先序线索二叉树遍历**：

| 函数 | 功能说明 |
|------|---------|
| `PreOrderNextNode(P)` | 找先序后继。若 `rtag == 1` 直接返回右线索；若有左孩子则返回左孩子，否则返回右孩子 |
| `PreOrderThread(T)` | 对先序线索二叉树进行先序遍历 |

**后序线索二叉树遍历**：

| 函数 | 功能说明 |
|------|---------|
| `PostOrderPreNode(P)` | 找后序前驱。若 `ltag == 1` 直接返回左线索；若有右孩子则返回右孩子，否则返回左孩子 |
| `RevPostOrderThread(T)` | 对后序线索二叉树进行逆向后序遍历 |

**核心特点**：
- 线索化后，遍历不需要递归或栈，时间复杂度 O(n)，空间复杂度 O(1)
- 中序线索二叉树的遍历最为常用，前驱和后继查找都很高效
- 先序和后序线索二叉树的遍历相对复杂，某些情况下仍需要栈辅助

#### 5.4 并查集 (DisjointSet.cpp)

**存储结构**：使用数组 `S` 表示森林，数组元素的值为负数时表示根结点（绝对值为该树的结点数），非负数时表示双亲结点的下标。

| 函数 | 功能说明 |
|------|---------|
| `Initial(S)` | 初始化并查集，所有元素设为 `-1`（各自为独立的集合，大小为 1） |
| `Find(S, x)` | 查找元素 `x` 所在集合的根结点，通过不断访问双亲结点直到找到根 |
| `FindPro(S, x)` | **路径压缩查找**：先找到根结点，再将查找路径上的所有结点直接指向根，压缩路径 |
| `Union1(S, root1, root2)` | **普通合并**：将 `root2` 的双亲设为 `root1`，简单但可能导致树退化为链表 |
| `Union2(S, root1, root2)` | **按秩合并**：将结点数少的树合并到结点数多的树中，保持树的平衡 |

**核心特点**：
- 路径压缩 + 按秩合并可以将时间复杂度优化到近似 O(1)
- 常用于解决连通性问题（如判断图中两点是否连通、最小生成树的 Kruskal 算法）
- 数组的负数表示根结点的设计非常巧妙，同时存储了根的信息和集合大小

### 6. 图 (5-Graph)

#### 6.1 邻接矩阵存储的图 (MGraph.cpp)

**存储结构**：使用二维数组 `Edge` 存储边的信息，`Edge[i][j] = true` 表示顶点 `i` 和 `j` 之间有边。`Vex` 数组存储顶点信息。

| 函数 | 功能说明 |
|------|---------|
| `InitMGraph(G)` | 初始化图，顶点数和边数设为 0，邻接矩阵所有元素设为 `false` |
| `InsertVertex(G, X)` | 插入顶点 `X`，顶点数加 1 |
| `InsertEdge(G, x, y)` | 插入边 `(x, y)`，无向图需同时设置 `Edge[x][y]` 和 `Edge[y][x]` |
| `Adjacent(G, x, y)` | 判断边 `(x, y)` 是否存在 |
| `Neighbors(G, x)` | 列出顶点 `x` 的所有邻接点 |
| `FirstNeighbor(G, x)` | 返回顶点 `x` 的第一个邻接点的顶点号，若无则返回 -1 |
| `NextNeighbor(G, x, y)` | 返回顶点 `x` 的邻接点 `y` 之后的下一个邻接点 |
| `BFS(G, v)` | **广度优先遍历**：从顶点 `v` 出发，使用队列辅助，逐层访问结点 |
| `BFSTraverse(G)` | 对整个图进行 BFS，处理非连通图的情况（遍历所有顶点作为起点） |
| `DFS(G, v)` | **深度优先遍历**：从顶点 `v` 出发，递归访问其未访问的邻接点，一条路走到黑 |
| `DFSTraverse(G)` | 对整个图进行 DFS，处理非连通图的情况 |
| `isTree(G)` | **判断无向图是否为一棵树**：需同时满足两个条件——边数等于顶点数减一（`E = V - 1`），且图是连通的（一次 BFS 能访问所有顶点） |

**核心特点**：
- 适合存储稠密图，空间复杂度 O(V²)
- 判断边是否存在时间复杂度 O(1)
- 获取某个顶点的所有邻接点需要遍历整行，时间复杂度 O(V)
- **BFS vs DFS**：BFS 适合求最短路径（无权图），DFS 适合拓扑排序、判断连通性等
- **isTree 算法**：先利用 `E = V - 1` 进行快速剪枝，再通过 BFS 验证连通性，确保图无环且连通

---

#### 6.2 邻接表存储的图 (ALGraph.cpp)

**存储结构**：使用数组（顶点表） + 链表（边表）的方式存储图。每个顶点对应一个链表，链表中存储该顶点的所有邻接点。

| 数据结构 | 说明 |
|---------|------|
| `ArcNode` | **边表结点**（弧结点），包含 `adjvex`（该弧指向的顶点在顶点表中的下标）和 `nextArc`（指向下一条弧的指针） |
| `VNode` | **顶点表结点**，包含 `data`（顶点信息）和 `firstArc`（指向第一条依附该顶点的弧的指针）。`AdjList` 是 `VNode` 数组的别名类型 |
| `ALGraph` | **图结构**，包含顶点表 `vertices`（`AdjList` 类型）、顶点数 `vexnum`、弧数 `arcnum` |

**当前实现**：
- 定义了完整的邻接表图数据结构
- `main` 函数中演示了手动构建有向图并打印邻接表的过程

**核心特点**：
- 适合存储稀疏图，空间复杂度 O(V + E)
- 获取某个顶点的所有邻接点只需遍历对应链表，时间复杂度 O(出度)
- 判断边是否存在需要遍历链表，时间复杂度 O(出度)
- 相比邻接矩阵，插入/删除边只需修改链表指针，更加灵活

### 7. 查找算法 (6-Search_Algorithm)

#### 7.1 顺序查找 (Search_Seq.cpp)

**存储结构**：查找表使用动态数组存储，通过 `tableLen` 记录当前表的长度。

| 函数 | 功能说明 |
|------|---------|
| `InitSSTable(T)` | 初始化查找表，分配最大容量为 `Maxsize` 的数组空间 |
| `Search_Seq(T, a)` | 在查找表 `T` 中顺序查找值为 `a` 的元素，找到返回下标，否则返回 -1 |

**算法思想**：从头到尾依次扫描每个元素，直到找到目标值或遍历完整个表。

**核心特点**：
- 时间复杂度：最好 O(1)（第一个就是），最坏 O(n)，平均 O(n)
- 空间复杂度：O(1)
- 对存储结构无要求，顺序存储和链式存储均可
- 适用于小规模数据或无序表

---

#### 7.2 二分查找 (BInary_Search.cpp)

**存储结构**：基于有序的顺序表，使用动态数组存储，查找前要求表中元素已按关键字有序排列。

| 函数 | 功能说明 |
|------|---------|
| `InitSSTable(T)` | 初始化查找表 |
| `Binary_Search(T, key)` | 在有序表 `T` 中二分查找值为 `key` 的元素，找到返回下标，否则返回 -1 |

**算法思想**：每次将查找区间对半分，通过比较中间元素与目标值的大小，决定继续在左半区间还是右半区间查找。

```
mid = (low + high) / 2
key == T[mid] → 找到
key <  T[mid] → high = mid - 1
key >  T[mid] → low = mid + 1
```

**核心特点**：
- 时间复杂度：O(log n)
- 空间复杂度：O(1)
- **仅适用于有序的顺序表**，不适用于链表（无法随机访问）
- 适合大规模数据的快速查找

---

#### 7.3 二叉排序树查找 (BST_Search.cpp)

**存储结构**：二叉排序树（BST），每个结点包含 `key`（关键字）、`lchild`（左孩子）、`rchild`（右孩子）。左子树所有结点值 < 根结点值 < 右子树所有结点值。

| 函数 | 功能说明 |
|------|---------|
| `BST_Search(T, key)` | 在二叉排序树 `T` 中查找关键字为 `key` 的结点，找到返回结点指针，否则返回 `nullptr` |

**算法思想**：利用 BST 的有序性质，若目标值小于当前结点则去左子树查找，大于则去右子树查找，等于则返回当前结点。

**核心特点**：
- 时间复杂度：最好 O(log n)，最坏 O(n)（退化为链表），平均 O(log n)
- 空间复杂度：O(1)（非递归实现）
- 同时支持高效的动态插入和删除
- 查找性能取决于树的平衡程度

---

### 8. 排序算法 (7-Sort_Algorithm)

#### 8.1 冒泡排序 (BubbleSort.cpp)

**算法思想**：重复遍历数组，每次比较相邻的两个元素，若顺序错误则交换。每一轮遍历将当前未排序部分的最大元素"冒泡"到正确位置。

| 函数 | 功能说明 |
|------|---------|
| `bubbleSort(array, n)` | 对数组 `array` 的前 `n` 个元素进行冒泡排序 |

**核心特点**：
- 时间复杂度：最好 O(n)，最坏 O(n²)，平均 O(n²)
- 空间复杂度：O(1)
- **稳定性**：稳定（相等元素不交换）
- 适合小规模数据或基本有序的数据

---

#### 8.2 直接插入排序 (InsertSort.cpp)

**算法思想**：将数组分为已排序和未排序两部分，每次从未排序部分取一个元素，插入到已排序部分的正确位置。

| 函数 | 功能说明 |
|------|---------|
| `insertSort(array, n)` | 对数组 `array` 的前 `n` 个元素进行插入排序 |

**核心特点**：
- 时间复杂度：最好 O(n)，最坏 O(n²)，平均 O(n²)
- 空间复杂度：O(1)
- **稳定性**：稳定
- 适合小规模数据或基本有序的数据，是希尔排序的基础

---

#### 8.3 简单选择排序 (SelectSort.cpp)

**算法思想**：将数组分为已排序和未排序两部分，每次从未排序部分选择最小元素，与未排序部分的第一个元素交换。

| 函数 | 功能说明 |
|------|---------|
| `selectSort(array, n)` | 对数组 `array` 的前 `n` 个元素进行选择排序 |

**核心特点**：
- 时间复杂度：最好、最坏、平均均为 O(n²)
- 空间复杂度：O(1)
- **稳定性**：不稳定（交换可能改变相等元素的相对顺序）
- 比较次数固定，但交换次数少（最多 n-1 次）

---

#### 8.4 指针学习 (Pointer.cpp)

**内容**：演示链表中的指针操作，包括 `->` 运算符和 `*` 运算符的使用。

| 操作 | 说明 |
|------|------|
| `head->data` | 通过箭头运算符访问指针指向的结构体成员 |
| `(*head).data` | 通过解引用和点运算符访问结构体成员 |
| `head->next->data` | 访问下一个结点的数据 |
| 链表遍历 | 使用 `while` 循环和 `current = current->next` 遍历链表 |

**核心特点**：
- `->` 是 `(*p).member` 的简写，两者等价
- 理解指针操作是学习链表、树、图等数据结构的基础

## 🎯 学习建议

### 1. 按顺序学习

建议按照目录编号顺序学习，从线性表开始，逐步过渡到树、图等复杂结构：

```
线性表 → 栈与队列 → 字符串 → 树与二叉树 → 图 → 查找算法 → 排序算法
```

### 2. 动手实践

每个 `.cpp` 文件都包含完整的测试代码，建议：
1. 先阅读代码理解算法思想
2. 自己尝试手写一遍
3. 修改测试用例，观察不同输入的输出结果
4. 尝试扩展功能（如在链表基础上实现排序）

### 3. 重点关注

- **链表操作**：指针的修改顺序是重点，建议画图理解
- **递归遍历**：二叉树的三种遍历方式是基础，必须熟练掌握
- **KMP 算法**：理解 `next` 数组的计算是难点，建议手动推导
- **图的遍历**：BFS 和 DFS 是图算法的基础

### 4. 调试技巧

- 使用 `cout` 输出中间结果，观察算法执行过程
- 对于指针问题，可以打印指针地址帮助定位问题
- 遇到段错误（Segmentation Fault），首先检查空指针访问
